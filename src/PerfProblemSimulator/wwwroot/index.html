🎮 Simulation Controls

🔥 CPU Stress ⓘ

Spawns worker threads running intensive cryptographic operations to stress CPU cores. Higher intensity uses more parallel threads.

📊 Memory Pressure ⓘ

Allocates pinned memory blocks filled with random data. Multiple allocations stack - mimics gradual memory leak.

🧵 Thread Pool Starvation ⓘ

Blocks worker threads using sync-over-async anti-pattern to simulate thread pool exhaustion. Server becomes unresponsive to new requests.

⏱️ Duration Note: .NET thread pool starts with ~CPU cores (e.g., 4-8) and grows at only 1-2 threads/second when starved. Impact is visible almost immediately since the initial pool is small.

🐌 Slow Requests ⓘ

Generates requests using sync-over-async patterns that intentionally take a long time. Ideal for CLR Profiler diagnosis in Azure.

⚠️ Note: Requests exceeding 30 seconds total time (queue + execution) are flagged as critical

❌ Failed Requests ⓘ

Generates HTTP 5xx errors by calling the load test endpoint with 100% failure rate. Creates authentic server errors for testing error monitoring.

📊 Where to see: Azure Portal → AppLens, Application Insights → Failures blade

💥 Application Crash ⓘ

Triggers a fatal crash to test Azure crash monitoring and diagnostics. Choose from various crash types including stack overflow and access violations.

⚠️ This will TERMINATE the app!
